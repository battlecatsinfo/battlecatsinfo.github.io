使用說明

篩選表達式、排序表達式

如1+1、2*3等，是用來篩選和排序貓咪的運算式，類似數學的表達式，在程式語言裡通常稱做expression。

  • 用在篩選上 篩選表達式將會算出一個結果，如果結果成立(true)，則會將該結果移除；如果是不成立(false)，則結果將會保留

    例如 range >= 400 篩選距離大於 400 的貓咪 例如 (dps > 40000) && (price <= 5000) 篩選DPS大於40000且價格小於5000的貓咪

  • 用在排序上 排序表達式將會算出一個數值，會以這個數值將篩選過的結果做由大到小的排序

    如果你沒有指定排序表達式，將會以貓咪的ID大小作為預設排序

    例如，輸入 dps 將會將dps由大到小排序

    例如，dps/cd 就是每秒的DPS產量(ADPS)

    例如，dps*range 就是DPS和距離的乘積

    如果看數值最小的排序，可以加上負號，如 -dps 就是從DPS最小排到最大

常數

有些預先定義的符號，稱為常數

如 RED 代表紅色敵人

如 AB_WAVE 代表波動能力

這些符號不用特別去記憶，點擊篩選按鈕就可以看到

變數

在上面的例子中，dps、cd、range 等都稱作變數，代表貓咪的某個資訊

以下是可以使用的變數清單(變數不分大小寫)

變數 解釋
id Cat ID
trait 針對屬性
imu 抗性
hp 血量
atk 攻擊力總和
attack 同 atk
range 攻擊距離或接觸點
attacks 攻擊頻率(秒)
attackf 攻擊頻率(f 幀數)
dps DPS(attack / attacks)
kb KB
cd 再生產
atktype 攻擊類型
rarity 稀有度
speed 移動速度
price 價格(二章、傳說)
cost 同price
thp 加上能力和效果的血量

tdps 加上能力和效果的DPS

backswing 後搖 收招時間

在使用 ‘tdps’ 和 ‘thp’ 時要注意終結使徒和終結魔女的加成很大，而且有些效果會疊加，如EVA零號機因終結使徒+很耐打使得 HP 乘 5 * 5 = 25 倍，造成破百萬的血量

以上數值都是以滿金寶、滿科技計算

另外 ( 和 ) 可以指定優先運算，如 -(1+2)*3 是 -9

含式

hasab: 有某種能力或效果，如 hasab(AB_WAVE) 是有波動能力的貓咪 hasres 是有某種耐性，如 hasres(RES_WEAK) 是有降攻耐性的貓咪 dpsagainst: 是對某屬性敵人的dps，其中除了常見的屬性(可以在篩選頁面選擇並複製篩選表達式)，TB_BEAST 是超獸，TB_BARON是超生命體，TB_EVA 是使徒，TB_WITCH 是魔女。而dpsagainst(AB_METAL) 比較特殊，它是計算對鋼鐵敵人造成的每秒傷害，所以沒有爆擊的機率會以1來計算，爆擊機率(含大傷、超傷、善攻)以雙倍計算，dps為兩者之和。

  • max: 最大值，如max(1, 2, 3)是3
  • min: 最小值，如min(1, 2, 3)是1
  • sqrt: 均方根值，如sqrt(81)是9
  • pow: pow(x, y)是x的次方
  • round: 四捨五入，如round(0.5)是1
  • random: 生成0-1之間的亂數(包含0，不包含1)
  • floor: 無條件捨去法，如floor(0.9)是0
  • ceil: 無條件進位法，如ceil(0.1)是1
  • sign: 如果輸入大於0，傳回1，如果輸入小於0，傳回-1，否則傳回0或-0
